# 演示脚本

1. 启动 Compose 并展示所有服务健康状态。
2. 上传论文并执行带程序 Citation ID 的单论文问答。
3. 发起信息不足请求，回答澄清问题后恢复原 Task。
4. 并行读取多篇论文并生成带证据的比较矩阵。
5. 从历史 Memory 和 Workspace 找回旧结论与脚本。
6. 使用 Writing Brief/Evidence Map 生成待审阅章节草稿。
7. 展示保留数字、公式、术语和引用的章节改写。
8. 注入 Tool 失败并展示最多两次的有界重规划。
9. 取消任务并从持久状态恢复。
10. 删除会话和文件，验证摘要、Embedding、缓存及检索结果失效。

录屏属于发布流程中的外部产物；仓库保存可重复的脚本和机器可读验收报告。
