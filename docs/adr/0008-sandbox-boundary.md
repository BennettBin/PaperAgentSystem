# ADR-0008: Sandbox 边界

**状态**: 已接受  
**日期**: 2026-06-19

## 背景

某些 Skill（如论文解析、代码执行）可能需要在隔离环境运行。但首版可能不实现真正的容器化沙箱。

## 决策

**MVP 阶段**:

- 代码执行默认禁止，返回明确的"不支持"错误
- LaTeX 渲染默认禁止，由前端使用 KaTeX
- PDF 解析在主进程，但严格验证输入

**未来扩展** (阶段 I03):

- 如需实现沙箱，使用 Docker 容器或 OCI runtime
- inputs 只读挂载
- 默认无网络访问
- 内存、CPU、磁盘配额限制
- 执行时间限制
- 输出扫描后返回

## 替代方案

1. **无限制执行**: 风险太高。
2. **禁止所有执行**: 功能受限。
3. **在主进程中隔离 (seccomp)**: 不可靠。
4. **容器化**: 资源开销大。

## 后果

### 优点

- ✅ MVP 快速交付
- ✅ 明确告知用户限制
- ✅ 不实现不可靠的假隔离

### 缺点

- ⚠️ 某些高级功能无法实现
- ⚠️ 未来需要投入沙箱实现

## 不变量

1. SandboxExecutor Port 返回明确的错误或真实隔离结果
2. 不存在隐式降级（Fake 沙箱）
3. 脚本文件默认不可执行
4. 用户代码不得修改系统配置

## 验收标准

- [ ] SandboxExecutor 对不支持功能返回明确错误
- [ ] 无隐式代码执行
- [ ] 文档清晰说明哪些功能支持沙箱
